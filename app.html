DEFINITIONS

Webpage:
A web page is a text document written in a language called HTML ( Hyper text Markup Language).

Coding:
Coding is what happens when programmers write text in a language that a computer can understand.

Tag:
A tag is a command. ALL tags have opening tags, but some do not have closing tags, these are called "void tags". See some examples of tags below.

Inline Tags vs. Block Tags:
Inline tags enclose content tightly. Block tags enclose content with a block creating a paragraph look.

Element:
An Element is everything in between a set of opening and closing tags.

Attribute:
An attribute is a property of an HTML element which provides additional information about the data within the element. For example: HREF ( Hypertext REFerence)

TAGS

Bold (b) Tag:
This tag makes the content in between the angled brackets bold.

Emphasis (em) Tag:
This tag makes the content in between the angled brackets italicized.

Anchor (a) Tag:
This tag defines a hyperlink. This tag will require an attribute which is href.
Dont know what a hyperlink is? Click here to find out!

Image (img) Tag:
This tag defines an image. This tag should always be followed by alternative (alt) text which will show if the image does not load.

CSS NOTES

CSS is what makes your web page look, well, pretty! But.... how do you use it? Here are a couple pointers.

1. You cam choose to write your css in your HTML or a seperate css file. Saving it in a seperate css file is the most efficient way though.

2. To point your css coding to the right direction you have to identify what you are giving stlye to. Here is how:

a. To point to a parragraph with a class, you start with "p." followed by the class name.

b. To point to a parragraph with an id, you start with "p#" followed by the class name.

c. To point to an image with a class, you start with "img." followed by the class name.

d. To point to an image with an id, you start with "img#" followed by the class name.

3. After pointing your code to the right direction, in curly brackets, type up your style!

4. It is important to NOT use repitition. It's simply a waste of time. That's when using "class" comes in handy.
